# Гидродинамический симулятор (Hydrodynamic Simulator)

Веб-приложение на **FastAPI** для загрузки и визуализации гидродинамических (фильтрационных) моделей нефтяных месторождений. Позволяет загружать `.DAT`-файлы сеточных моделей, парсить сетку и инициализацию (`.mgrid`/`.minit`), вычислять объёмы ячеек и строить карты средних значений проницаемости, пористости, насыщенности и подвижных запасов.

---

## Стек

- **Backend:** Python, FastAPI, Starlette
- **Templates:** Jinja2, Bootstrap 5 (CDN)
- **Научные библиотеки:** NumPy, SciPy (ConvexHull), Matplotlib
- **Файловый формат:** Eclipse-style ключевые слова (SPECGRID, COOR, ZCOR, PORO, K_X, SWAT и др.)

---

## Установка и запуск

### Требования

- Python 3.10+
- Виртуальное окружение (рекомендуется)

### Установка зависимостей

```bash
python -m venv venv
source venv/bin/activate      # Linux/macOS
venv\Scripts\activate         # Windows
pip install -r requirements.txt
```

### Запуск сервера

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Приложение будет доступно по адресу: [http://localhost:8000](http://localhost:8000)

## Использование

### 1. Загрузка модели (`POST /upload`)
Загрузите `.DAT`-файл модели. Приложение автоматически:
- Распарсит ключевые слова: `SPECGRID`, `TITL`, `UNIT`, `MODE`, `INCL` и др.
- Создаст директорию `uploads/<имя_файла>/`
- Сохранит файл и перечислит ожидаемые include-файлы

### 2. Загрузка include-файлов (`POST /upload_include`)
Загрузите файлы `.mgrid` (сетка) и `.minit` (инициализация) в ту же директорию модели.

### 3. Расчёт (`GET /calculate`)
- Читает `.mgrid` и `.minit`
- Парсит координаты узлов (`COOR`), Z-координаты углов (`ZCOR`)
- Вычисляет объём каждой ячейки через `ConvexHull`
- Вычисляет нефтенасыщенность: `SOIL = 1 - SWAT`
- Выводит таблицу с размерами сетки, значениями свойств

### 4. Визуализация (`GET /plot`)
Строит карты средних (или суммарных) значений по оси Z:
- **Проницаемость** (K_X, мД)
- **Пористость** (PORO, д. ед.)
- **NTG** — песчанистость (NTOG, д. ед.)
- **Водонасыщенность** (SWAT, д. ед.)
- **Нефтенасыщенность** (SOIL, д. ед.)
- **Объёмы** (VOLU)
- **Подвижные запасы нефти**: `VOLU × NTG × PORO × SOIL`

Изображения сохраняются в `static/plots/<имя_модели>/`.

### 5. Удаление файлов (`POST /delete_file`)
Удаляет файл из директории текущей модели.

## Формат файлов

### `.DAT` (основной файл модели)
Ключевые слова и данные (Eclipse-подобный формат):
```
SPECGRID
20 30 10 1 FLOAT /
TITL
My Reservoir Model /
INCL
'model.mgrid'
INCL
'model.minit'
```

Поддерживается нотация повторений: `3*0.25` → `0.25 0.25 0.25`.

### `.mgrid` (сетка)
Содержит ключевые слова `COOR` (координаты узлов, 6 чисел на узел) и `ZCOR` (Z-координаты 8 углов каждой ячейки).

### `.minit` (инициализация)
Содержит ключевые слова `PORO`, `K_X`, `NTOG`, `SWAT` и др.

## API endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` | Главная страница, список файлов и параметров |
| POST | `/upload` | Загрузка `.DAT`-файла модели |
| POST | `/upload_include` | Загрузка include-файлов (`.mgrid`, `.minit`) |
| POST | `/delete_file` | Удаление файла из модели |
| GET | `/calculate` | Расчёт объёмов и свойств |
| GET | `/plot` | Построение карт средних значений |

## Состояние

Всё состояние хранится в глобальном словаре `MODEL_STATE` в памяти. **При перезапуске сервера данные теряются.** Персистентность не реализована.
